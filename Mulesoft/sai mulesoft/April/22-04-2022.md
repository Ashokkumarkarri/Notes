## Anypoint Studio & Mule Runtime Versions
- Anypoint Studio is the IDE used for MuleSoft development.
- Latest Anypoint Studio version discussed: 7.11.
- Important distinction: the Anypoint Studio (IDE/application) version is different from the Mule runtime version — don't confuse the two.
- Mule runtime version currently being worked on: 4.4.4.

## Secure Properties & Encryption
- Example: a project used the AES algorithm for encryption with a 16-digit secure key.
- Syntax to read an encrypted (secure) property value at the config level: `${secure::keyName}`
- Syntax to read an encrypted (secure) property value inside Transform Message (DataWeave): `p('secure::keyName')`
- Syntax to read a non-encrypted property value: `${keyName}`

## CloudHub — Notifications and Scheduling
- To trigger email notifications from CloudHub, use the CloudHub notification connector.
- For scheduling jobs / calling APIs, HTTP Request components are used; the domain and application name should match.
- HTTP Request can be used to call other API-based applications.

## Message Queuing — Queue vs. Exchange
- Async messaging follows a pub/sub model.
- Discussion covered the difference between a queue and an exchange, and how to create queues and exchanges on the platform.
  *(Note: the specific product name was garbled in the audio transcript — based on the queue/exchange terminology used, this is most likely RabbitMQ.)*
- FIFO = First In, First Out — messages are processed in the order they were published to the queue (the first subscriber to the queue receives them first).
- A normal queue allows multiple consumers to pick up messages simultaneously.

## Object Store
- Object Store is used to store data internally rather than fetching it repeatedly from an external system.
- Maximum payload size that can be pushed into Object Store: 10 MB.
- Maximum retention: data can be stored in Object Store for up to 30 days.
- There is a "persistent" option/checkbox — if checked, data survives application downtime (won't be lost); if not checked, data is lost when the app goes down.
- Object Store otherwise runs on runtime memory.

## DataWeave / Transform Message Functions
- Interview-relevant DataWeave functions mentioned: `map`, `sizeOf`, `splitBy`, `replace`, `pluck`, `flatten`, and date functions.
- Recommendation: know a minimum of 10 DataWeave functions thoroughly for interviews — this is a commonly asked topic.

## MUnit Testing
- MUnit is used to test Mule applications internally (without external dependencies).
- Test cases are written and stored under `src/test/munit`.

## POM File & Dependencies
- Project dependencies are stored/managed in the `pom.xml` (POM) file.

## Property Files
- Property files are used to secure sensitive information (via secure properties).
- Global configuration uses property placeholders so the application can point to different environments — e.g., switching a property file/config to point to production vs. test.

## CloudHub Platform Components
- Components discussed for CloudHub deployments: Exchange, API Manager, monitoring, Object Store, and Scheduler.
