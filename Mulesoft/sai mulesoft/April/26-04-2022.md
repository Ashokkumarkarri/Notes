## Pushing an Anypoint Studio Project to Bitbucket
- Example project used: an Anypoint Studio project named "Dummy".
- Before committing/pushing a project, you must first create an empty repository in Bitbucket.
- A freshly created empty Bitbucket repository has no files yet — a typical Mule project structure includes `src/main`, `src/test`, and the `pom.xml` (POM) file; a brand-new empty repo won't yet contain files like `artifact.json`.

## Bitbucket Branching Strategy
- The default branch created with a new repository is `master`.
- Standard practice across organizations: the `master` branch is reserved only for production deployment — it should always mirror what's running in production.
- Code that is still being tested/developed should never be pushed directly to `master`.
- For an empty/new project (like the "Dummy" project) that isn't ready for production, create a separate new branch (e.g., `develop`) for testing and ongoing development work.

## Creating and Cloning a Branch
- Steps to create a new branch in Bitbucket:
  1. Go to the repository's **Branches** section.
  2. Click **Create branch** (top-right corner).
  3. Give the branch a name (e.g., `develop`).
- To clone a specific branch locally using Git Bash:
  1. Right-click in the target folder → **Git Bash Here**.
  2. Copy the repository's clone URL from Bitbucket.
  3. Run: `git clone -b <branch-name> <repository-url>` — e.g., `git clone -b develop <repo-url>`
  4. The `-b` flag specifies which branch to clone (without it, the default `master` branch would be cloned instead).
  5. Enter Bitbucket credentials when prompted to authenticate the clone.
