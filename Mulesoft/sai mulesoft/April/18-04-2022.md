## Circuit Breaker Pattern with Dead Letter Queue (DLQ)
- Created a new queue named **"Circuit DLQ"**, and bound it to the main queue ("Circuit") as its assigned **Dead Letter Queue (DLQ)**.
- Configured **Max Delivery = 2** on the main queue: this means a given message will be subscribed/attempted from the main queue only 2 times. On the 3rd attempt, the message is deleted from the main queue and pushed into the DLQ instead.
- Demonstrated end-to-end: pushed a test message, observed it being picked up and retried; after exceeding Max Delivery attempts, the message moved from the main queue into the DLQ.
- Discussed how **Circuit Breaker threshold** and **Anypoint MQ Max Delivery/redelivery timing** relate to each other:
  - Example: Circuit Breaker **threshold = 5** — meaning if the flow encounters 5 errors, the circuit "trips" and the flow stops attempting further calls.
  - Example: Anypoint MQ **Max Delivery = 2** — the same message is redelivered/re-attempted 2 times from the queue before being routed to the DLQ.
  - Key rule: the **relay/redelivery wait value should always be set higher than the threshold value** (example given: with threshold = 5, the relay/wait setting should be around 10) — otherwise the circuit breaker and the MQ redelivery timing can conflict.
  - Example retry interval used in the walkthrough: the flow waits 2 minutes between retry attempts once the circuit breaker trips.
  - Walked through the resulting behavior step by step: with threshold = 5, the flow will pick up and retry the message multiple times (up to the threshold count) before tripping; once tripped, it pauses for the configured wait time, then resumes, and the message is eventually pushed to the DLQ once Max Delivery is exceeded.
