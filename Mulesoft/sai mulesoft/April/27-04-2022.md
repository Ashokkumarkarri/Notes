## CI/CD Pipeline Stages
- Pipeline rules/checks before deployment:
  - No commented-out code is allowed in the codebase.
  - Sensitive information must be encrypted in the property file — not everything should be checked in as plain text.
  - These checks are enforced in the validation stage of the pipeline.
- A later stage in the pipeline enforces code coverage — code coverage must be more than 80%, otherwise the pipeline will not allow the application to be deployed to runtime.
- The final step of the pipeline is deployment to the runtime stage.
- If all stages are followed correctly, the pipeline's generated script validates the code end-to-end.

## Bitbucket Branching Strategy (Across Technologies)
- Code repository used: Bitbucket.
- Branching model: `master`, `develop`, and `feature` branches.
- `master` branch code is only for production deployment; the production code always lives on `master`.
- Remaining/in-progress code is worked on in development branches.
- This branching approach is not specific to Mule — it applies the same way across all technologies used by the organization.
- Pipelines scan and validate the complete codebase before allowing deployment.

## Anypoint MQ Limitations and Solace as an Alternative
- Anypoint MQ limitations:
  - Cannot push a payload larger than 10 MB.
  - Messages can be retained for a maximum of 7 days.
- Solace is presented as an alternative to Anypoint MQ:
  - Solace gives more control over messages sitting in the queue.
  - Solace supports pushing a much larger payload size than Anypoint MQ.
  - The JMS connector is used to integrate with Solace.
  - Usage is conceptually similar to Mule/Anypoint MQ — no drag-and-drop components; a queue is created, and messages are published to that queue.
  - Where Anypoint MQ would be used, Solace can be used in its place by publishing to a Solace queue instead.

## Doubts / Q&A

**Q: Will the MuleSoft certification exam still be relevant, or could the questions have changed since others cleared it?**
- A: The certification is free of cost (no fee required to attempt it).
- The question set had reportedly last changed about 2 months prior; those who cleared it recently faced the same set of questions.
- Since the question bank can change, students were encouraged to attempt the certification as soon as possible while the current question set is still active — waiting could mean a wait of 1–2 months for a new set if it changes.
- On passing, the certificate can be added to the resume (with the certification logo and a title such as "Certified Developer"); multiple certifications can be listed if held.
- Additional resume guidance: candidates can also reference articles/publications they've written and mention their previous client/project experience.
