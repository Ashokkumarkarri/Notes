## Transform Message — Defining Sample Input/Output Metadata
- In the **Transform Message** component, you can define concrete sample metadata for both the input and the output instead of mapping blind.
- To set input metadata: click **Define Metadata** → **Add** → give the sample a name → load a sample file (e.g., an XML file saved locally on your machine) as the example input structure.
- The sample XML used in the demo had two root-level elements (audio was unclear on the exact tag names).
- The same approach applies to the output side: click **Add**, give it a name (e.g., "output"), and load a sample file (e.g., a JSON file) representing the structure the target system expects.
- Purpose: loading sample input/output data lets Anypoint Studio generate the DataWeave mapping against a concrete, known schema rather than an undefined structure.
